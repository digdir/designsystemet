Nokre gangar må du kanskje bytte ut ein komponent med ein anna, for eksempel `Link` skulle brukt `NextLink` i staden for `a`-taggen.
Det er her `asChild` kjem inn i biletet.

<CodeSnippet language='ts'>
  {`
  import { Link } from '@digdir/design-system-react';
  import { Link as NextLink } from 'next/link';

  <Link asChild>
    <NextLink href='https://www.digdir.no'>Lenke til digdir.no</NextLink>
  </Link>
`}
</CodeSnippet>

I kodesnutten over blir `Link`-komponenten rendra som ein `NextLink`-komponent. Når dette skal ut i DOMen, så er det kun eit element som blir rendra ut.
Dette skjer ved hjelp av Radix sin `Slot`-komponent. [1]

`Slot` merger sine props ned på den komponenten som ligger som direkte barn. I tilfellet over så vil `Link` sine props bli lagt til på `NextLink`-komponenten.
Når du bruker `asChild`, så kan du ikkje ha meir enn eit barn, men du kan ha så mange du vil inne i det barnet.

<CodeSnippet language='ts'>
  {`
  <>
    /* Dette kaster ein error */
    <Link asChild>
      <Icon />
      <NextLink href='https://www.digdir.no'>Din lenke</NextLink>
    </Link>

    /* Dette går fint */
    <Link asChild>
      <NextLink href='https://www.digdir.no'>
        <Icon />
        Din lenke
      </NextLink>
    </Link>

  </>
`}
</CodeSnippet>

## Kvifor bruke asChild?

Me har tidlegare brukt ein `as` prop for å rendre som andre element, men når du bruker denne, så får du ikkje typesafety.
`Slot` fikser dette ved at du legger alle props på barnet av komponenten, og du får dermed typesafety.

<CodeSnippet language='ts'>
  {`
  <Link asChild>
    <NextLink href='https://www.digdir.no' onClick={() => {}}>
      <Icon />
      Din lenke
    </NextLink>
  </Link>
`}
</CodeSnippet>

Alt av klassenavn, aria-attributer og andre props som `Link` har, vil bli lagt til på `NextLink`-komponenten.
Dette betyr at me kan tilby god tilgjengelegheit, samtidig som du kan bruke andre komponenter som du ønsker.

## Event handlers

Dersom ein prop starter med `on` (fks. `onClick`), så er det sett på som ein event handler.
Når `Slot` merger props, vil det lage ein ny funksjon som kallar alle event handlers som er definert på `Link` og `NextLink`, men funksjonen som ligger på `NextLink` vil bli kalla først.
Dette betyr at dersom du stopper eit event på din komponent, vil eventet på `Slot`-komponenten ikkje bli kalla.

Dersom ein av event handlerane er avhengig av `event.defaultPrevented`, så må du passe på at rekkefølga er korrekt. [2]

<CodeSnippet language='ts'>
  {`
  <Link
    asChild
    onClick={(event) => {
      if (!event.defaultPrevented)
        console.log('Not logged because default is prevented.');
    }}
  >
    <NextLink onClick={(e) => e.preventDefault()}>Din lenke</NextLink>
  </Link>
`}
</CodeSnippet>
