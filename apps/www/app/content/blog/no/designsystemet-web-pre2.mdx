## Teste i eksisterande applikasjon

Me har oppdatert React-pakken til å bruke `@digdir/designsystemet-web` som underliggande rammeverkuavhengig komponentbibliotek.
Dette betyr at nokre komponentar som i dag er implementert i React, kan opplevast annleis sidan dei no kjem frå web-pakken.
Gjerne test desse og kom med tilbakemeldingar.

Desse er:
- Popover
- Dropdown
- Tooltip
- Dialog
- Pagination
- Breadcrumbs
- Tabs
- ToggleGroup
- Field

Gjerne sjekk alt av tester og konsoll for å sjå etter feil eller uventa oppførsel. 

`ToggleGroup` har fått ny intern struktur, slik at den no ikkje brukar `<button>`.
Den nye strukturen er:
```html
<fieldset class="ds-toggle-group" data-toggle-group="Tekstjustering">
  <label>
    <input type="radio" name="alignment-two" value="left" checked />
    Venstestilt
  </label>
  <label>
    <input type="radio" name="alignment-two" value="center" />
    Midtstilt
  </label>
  <label>
    <input type="radio" name="alignment-two" value="right" />
    Høyrestilt
  </label>
</fieldset>
```

## Teste i andre rammeverk eller rein HTML/JS

For å teste pakken utan React kan du installere `@digdir/designsystemet-web` og `@digdir/designsystemet-css` i ditt prosjekt.
Deretter brukar du komponentane og funksjonaliteten direkte.

Det blir også generert typar for diverse rammeverk.
Desse får du ved å legge til i din `tsconfig.json`:

```json
{
  "compilerOptions": {
    "types": ["@digdir/designsystemet-web"]
  }
}
```
