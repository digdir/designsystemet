Vi har nå lansert `@digdir/designsystemet-web` pakken.

Denne pakken fungerer nå som et fundament for Designsystemets interaktive komponenter og hjelpere til bruk av native webteknologier.

Denne første utgivelsen fokuserte på å gi et solid fundament for vår React-pakke (`@digdir/designsystemet-react`) som nå er bygget på toppen av web-pakken.

## Hva er inkludert?


### `<ds-breadcrumbs>`
Skjuler/viser automatisk `aria-label` på desktop/mobil og `aria-current="page"` på siste lenke i listen. Ingen API.

```html
<ds-breadcrumbs class="ds-breadcrumbs" aria-label="Du er her:">
  <a href="#none" aria-label="Tilbake til nivå 3">
    Nivå 3
  </a>
  <ol>
    <li><a href="#none">Nivå 1</a></li>
    <li><a href="#none">Nivå 2</a></li>
    <li><a href="#none">Nivå 3</a></li>
  </ol>
</ds-breadcrumbs>
```

### `<ds-error-summary>`
Tar automatisk fokus når den er synlig og setter `aria-labelledby` til det første barnets overskrift. Ingen API.

```html
<ds-error-summary class="ds-error-summary">
  <h2>Oppsummering</h2>
  <ul>
    <li><a href="#none">Feil 1</a></li>
    <li><a href="#none">Feil 2</a></li>
    <li><a href="#none">Feil 3</a></li>
  </ul>
</ds-error-summary>
```

### `<ds-field>`
Kobler sammen input, etiketter og feilmeldinger.

```html
<ds-field class="ds-field">
  <label>Etikett</label>
  <input type="text" placeholder="Plassholder" class="ds-input" />
  <div class="ds-validation-message" data-field="validation">
    Dette er en feilmelding.
  </div>
</ds-field>
```

### `<ds-pagination>`
Implementerer paginering, fyller knapper med tekst.
Du kan bruke både `<a>` og `<button>` elementer inne i pagineringen.

Hvis du ikke sender noen attributter, kan du implementere din egen logikk for gjeldende side og totalt antall sider.

```html
<ds-pagination class="ds-pagination" aria-label="Bla i sider:" data-href="?page=%d" data-current="2" data-total="100">
  <ol>
    <li><a>1</a></li>
    <li><a>2</a></li>
    <li><a>3</a></li>
    <li><a>4</a></li>
  </ol>
</ds-pagination>
```

### `<ds-suggestion>`
Utvider `u-combobox` fra u-elements. Se dokumentasjonen for [u-combobox](https://u-elements.github.io/u-elements/elements/u-combobox).

```html
<ds-field class="ds-field">
  <label>Etikett</label>
  <ds-suggestion class="ds-suggestion">
    <input type="search" class="ds-input" />
    <del aria-label="Fjern innhold"></del>
    <u-datalist>
      <u-option value="option-1">Option 1</u-option>
      <u-option value="option-2">Option 2</u-option>
      <u-option value="option-3">Option 3</u-option>
    </u-datalist>
  </ds-suggestion>
</ds-field>
```


### `<ds-tabs>`
Utvider `u-tabs` fra u-elements. Se dokumentasjonen for [u-tabs](https://u-elements.github.io/u-elements/elements/u-tabs).

```html
<ds-tabs class="ds-tabs">
  <ds-tablist>
    <ds-tab>Tab 1</ds-tab>
    <ds-tab>Tab 2</ds-tab>
    <ds-tab>Tab 3</ds-tab>
  </ds-tablist>
  <ds-tabpanel>Panel 1</ds-tabpanel>
  <ds-tabpanel>Panel 2</ds-tabpanel>
  <ds-tabpanel>Panel 3</ds-tabpanel>
</ds-tabs>
```

### `data-toggle-group`
Dette er implementert annerledes enn `ToggleGroup` i React-pakken.

En observer vil se etter `[data-toggle-group]` og legge til riktig pilnavigasjon pluss Enter-taststøtte.

```html
<fieldset class="ds-toggle-group" data-toggle-group="Tekstjustering" data-variant="secondary">
  <label>
    <input type="radio" name="alignment-two" value="left" checked />
    Venstrejustert
  </label>
  <label>
    <input type="radio" name="alignment-two" value="center" />
    Sentrert
  </label>
  <label>
    <input type="radio" name="alignment-two" value="right" />
    Høyrejustert
  </label>
</fieldset>
```

### `data-tooltip`
Bruker et enkelt element for å rendre ved siden av elementer med `data-tooltip`-attributt.
Setter også automatisk `aria-label` eller `aria-description` etter behov.

```html
<button data-placement="left" data-tooltip="left" class="ds-button">left</button>
```
### `data-clickdelegatefor`
Brukes for å delegere klikkhendelser. For eksempel kan du bruke dette for å delegere klikkhendelser fra et overordnet element til underordnede elementer som legges til dynamisk.

```html
<div class="ds-card" data-clickdelegatefor="target">
  <a id="target" href="https://example.com" rel="noopener">Gå til eksempel</a>
  <span>Klikke på dette kortet vil åpne eksemplet i en ny fane</span>
</div>
```

### `readonly`
Brukes for å fikse `readonly`-støtte på `select`- og `input`-elementer. Legg til `aria-readonly="true"` for å gjøre elementet readonly, noe som betyr at det ikke kan redigeres av brukeren eller utløse noen endringshendelser.

```html
<select aria-readonly="true">
  <option value="1">Option 1</option>
  <option value="2">Option 2</option>
  <option value="3">Option 3</option>
</select>
```


### Polyfills

#### invokers-polyfill
Vi legger automatisk til [invokers-polyfill](https://www.npmjs.com/package/invokers-polyfill/v/0.5.2), noe som betyr at du får støtte for `command` og `commandfor`.

#### `<dialog>`
Støtte for [closedby="any"](https://developer.mozilla.org/en-US/docs/Web/API/HTMLDialogElement/closedBy#any).
