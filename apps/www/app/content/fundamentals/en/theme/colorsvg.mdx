If the graphics should look exactly the same regardless of light or dark mode, and use the colours defined in the design tool, you do not need to follow the guidance below.

However, if you want to control colours and scaling via code (for example using design tokens), you should be added graphics as SVGs and they should be structured in a specific way.

There are mainly two methods to do this:

- SVG inline in the html
- `<use>`, coupled with `<symbol>` for reuse

Both works as intended. While `<use>` is preferred when the same graphics is used multiple places without having to duplicate code, inline SVG is often easiest when the grphics is only used once.

Both colour and size is addressable with CSS when SVG is used inline or with `<use>`. For this to work, the SVG need to be set up correctly, for example including `viewBox`.

Other methods for adding graphics, i.e. `<img src="...svg">`, canvas, CSS-drawing or icon fonts, does not give the same colour controll using design tokens, these methods are therefore not included in this guide.

## Basic setup

To make an SVG easy to reuse and style, you should prepare the file as follows:

- wrap the graphics in `<symbol>`
- give the symbol an id
- define a viewBox on the symbol

```SVG
<svg xmlns="http://www.w3.org/2000/svg">
  <symbol id="ds-logo" viewBox="0 0 40 40">
    <path d="..." />
    <path d="..." />
    <path d="..." />
    <path d="..." />
  </symbol>
</svg>
```
The file can then be used in HTML like this:
```HTML
<svg height="1rem" width="1rem">
  <use href="logo.svg#ds-logo"></use>
</svg>
```

<Alert data-color='info'>
  <h3 class="ds-heading" data-size='xs'>Tip: Scaling SVG</h3>
  <br />
  SVG scales easiest when:
  - viewBox is set on `<symbol>` or `<svg>`
  - size is controlled using width and height on the wrapping html element
  <br />
  If the SVG does not scale as expected, it will often be because:
  - viewBox is missing
  - or the proportions is wrong
</Alert>

## Practical implementation

What you need to adjust further depends on what you want to achieve.

### Single-colour graphics (typical for certain logos and icons)

For a single-colour graphics, you can control the colour either by inheriting the current text colour or by applying a class or id.

If you want all SVG graphics to use the current text colour by default, you can use currentColor. Remove all colour values from the SVG graphics and add a rule for currentColor, so the illustration inherits the surrounding text colour. You may need to do the same for stroke.
```CSS
svg {
    fill: currentColor;
}
```

If you do not want to apply a global rule, you can instead set currentColor directly on the fill and/or stroke attributes inside the SVG file itself. This approach is used for icons in the design system, for example in buttons.
```SVG
<svg xmlns="http://www.w3.org/2000/svg">
  <symbol id="ds-logo" viewBox="0 0 40 40">
    <path d="..." fill="currentColor"/>
    <path d="..." stroke="currentColor"/>
    <path d="..." fill="currentColor"/>
    <path d="..." stroke="currentColor"/>
  </symbol>
</svg>
```

If you need more control, you can manage colours via a class or id. Remove all colours from the SVG and add a class to the outer `<svg>` element, for example:
```HTML
<svg height="1rem" width="1rem" class="iconcolour">
  <use href="logo.svg#ds-logo"></use>
</svg>
```

With a corresponding token:
```CSS
.iconcolour {
    color: var(--ds-color-text-default);
}
```

### Multi-colour graphics (typical for brand elements and small illustrations)

For graphics with multiple colours, you can control colours directly on fill and stroke for each path inside the SVG file.
```SVG
<svg xmlns="http://www.w3.org/2000/svg">
  <symbol id="ds-logo" viewBox="0 0 40 40">
    <path d="..." fill="var(--ds-color-logopart1)"/>
    <path d="..." fill="var(--ds-color-logopart2)"/>
    <path d="..." fill="var(--ds-color-logopart3)"/>
    <path d="..." fill="var(--ds-color-logopart4)"/>
  </symbol>
</svg>
```
With corresponding tokens. In this example, the graphics uses greyscale in dark mode:
```CSS
:root {
    --ds-color-logopart1: #1E98F5;
    --ds-color-logopart2: #68707C;
    --ds-color-logopart3: #E5AA20;
    --ds-color-logopart4: #F45F63;
    @media (prefers-color-scheme: dark) {
        --ds-color-logopart1: #A3A3A3;
        --ds-color-logopart2: #68707C;
        --ds-color-logopart3: #C2C2C2;
        --ds-color-logopart4: #8A8A8A;
    }
}
```
You can of course reference colours directly from the design system, or use dedicated tokens if you have defined them for illustrations.

## Accessibility considerations
To ensure the graphics is accessible, you need to add a few attributes to the outer `<svg>` element.

### Decorative graphics
Hide the graphic from screen readers by setting its role to presentation and marking it as hidden:
```HTML
<svg height="1rem" width="1rem" class="logocolour" role="presentation" aria-hidden="true">
  <use href="logo.svg#ds-logo"></use>
</svg>
```

### Meaningful graphics
If the graphic conveys meaning, give it the role of an image and provide a descriptive label:
```HTML
<svg height="1rem" width="1rem" class="logocolour" role="img" aria-label="Design system logo with colours defined via variables">
  <use href="logo.svg#ds-logo"></use>
</svg>
```
For more complex graphics and illustrations, you may need more detailed descriptions. There are also ways to embed such descriptions directly within the SVG, but that is outside the scope of this guide.


## Inline graphics

Using `<use>` allows you to avoid repeating the same SVG code in multiple places. If the illustration is only used once on a page, it is perfectly acceptable to include it inline instead.

```HTML
<svg width="80" height="80" viewBox="0 0 40 40" xmlns="http://www.w3.org/2000/svg">
  <path d="..." fill="var(--ds-color-logopart1)"/>
  <path d="..." fill="var(--ds-color-logopart2)"/>
  <path d="..." fill="var(--ds-color-logopart3)"/>
  <path d="..." fill="var(--ds-color-logopart4)"/>
</svg>
```

## Vanlige problemer og hva du bør sjekke

Common errors for when the graphics does not look right:

- **SVG does not scale or is the wrong size, check that**
  - `viewBox` is set
  - both `width` and `height` is defined on `<svg>` or via CSS

- **Colours does not change, make sure**
  - the SVG is inline in the DOM (not via `<img>`)
  - `fill` and `stroke` is not hard coded colour values, like hex
  - the CSS-variables (`var(...)`) is defined and accessible where the SVG is used

- **Graphics is single coloured when expected to be multicoloured**
  - check if `currentColor` is used on paths
  - look for global styles that addresses all SVGs

- **Graphics does not show at all**
  - Make sure `href` iside `<use>` have the right file path and `id` (i.e. `file.svg#id`)
  - Check the SVG file for `<symbol>` and the correct `id`
