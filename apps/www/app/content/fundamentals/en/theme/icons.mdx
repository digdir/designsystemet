## Choose your own icon set

With Designsystemet, you choose which icon set to use. Some use Nav's icon set, while others choose alternatives that better match their visual profile. It is entirely up to you and what works best for your organization.

## Swap icon set in Figma

When you download our Community file, it will contain links to our internal version of Aksel's icon set. Whether you use Aksel or another icon library, you need to reconnect the icons to an icon set that is stored in your Figma organization.

### Option 1: Set up your own copy of Aksel Icons

1. Download [Aksel icons (Figma)](https://www.figma.com/community/plugin/1221057403873975172/aksel-icons)
2. Publish the icon set as a library in Figma
3. Use the [Swap library (Figma)](https://help.figma.com/hc/en-us/articles/4404856784663-Swap-style-and-component-libraries) function to replace the icons in the Community file with your local icons

### Option 2: Set up another icon library instead of Aksel Icons

Icons are often part of the visual identity of the organization that will use the design system. That means Aksel's icons will not always be a good fit. If you want to use another library, there are a couple of things to keep in mind.

The easiest way to replace the icons is to use [Swap library (Figma)](https://help.figma.com/hc/en-us/articles/4404856784663-Swap-style-and-component-libraries), but this function uses the component name to determine which icon should be replaced. One simple way to handle this is to rename the icons you want to use so they match the names Aksel uses. After the swap, you can change them back to their original names. Below you will find two tables with icon names and examples. For the meaningful icons, we recommend choosing icons that are as similar as possible. The non-meaningful icons are placeholder icons used in places such as buttons. In those cases, it is less important which icon you replace them with.

When you open Swap library, you will see many more icons than the ones listed below. These are icons used in examples and do not have any functional consequences if they are not replaced. They will be removed in the next version of the Community file.
