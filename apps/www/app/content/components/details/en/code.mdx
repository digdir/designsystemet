## Usage
Details is a native `<details>` element.
We polyfill a bug in Firefox when used with Android Talkback screen reader to announce state and role correctly.

```html
<!-- without frame -->
<details class="ds-details">
  <summary>
  Details heading text
  </summary>
  <div>
  Details content
  </div>
</details>

<!-- with frame -->
<div class="ds-card">
  <details class="ds-details">
    <summary>
    Details heading text
    </summary>
    <div>
    Details content
    </div>
  </details>
</div>
```

## CSS variables and data attributes

## React

```tsx
/* Without frame */
import { Details } from '@digdir/designsystemet-react';

<Details>
  <Details.Summary>Details heading text</Details.Summary>
  <Details.Content>Details content</Details.Content>
</Details>

/* With frame */

import { Details, Card } from '@digdir/designsystemet-react';

<Card>
  <Details>
    <Details.Summary>Details heading text</Details.Summary>
    <Details.Content>Details content</Details.Content>
  </Details>
</Card>
```
