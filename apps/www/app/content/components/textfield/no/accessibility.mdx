## Dette må du sjekke selv

**Feltet skal ha en synlig label.**

Skjermlesere skal presentere ledetekst, hjelpetekst og feilmeldinger når feltet får fokus.

Du kan bruke [Field](/no/components/docs/field) for å koble sammen label, beskrivelse og feilmeldinger.

<span data-size="sm">
WCAG: [1.3.1 Informasjon og relasjoner](https://www.uutilsynet.no/veiledning/131-informasjon-og-relasjoner/1234)
</span>

<Divider/>

\
**Feltet skal ha et tydelig tilgjengelig navn som samsvarer med den synlige label.**

Det tilgjengelige navnet skal inneholde den synlige teksten som beskriver feltet. Dette gjør at brukere av talegjenkjenning og skjermlesere kan referere til feltet med samme navn som vises på skjermen.

<span data-size="sm">
WCAG: [2.5.3 Ledetekst i navn](https://www.uutilsynet.no/veiledning/253-ledetekst-i-navn/1256)
</span>

<Divider/>

\
**Obligatoriske felt skal være tydelig merket.**

Brukeren skal enkelt kunne forstå hvilke felt som må fylles ut. Følg [mønsteret for obligatoriske og valgfrie skjemafelt](https://designsystemet.no/no/patterns/required-and-optional-fields).

<span data-size="sm">
WCAG: [3.3.2 Ledetekster eller instruksjoner](https://www.uutilsynet.no/veiledning/332-ledetekster-eller-instruksjoner/1263)
</span>

<Divider/>

\
**Feilmeldinger skal være tydelig knyttet til riktig felt og forklare hvordan feilen kan rettes.**

Feilmeldingen skal gjøre det klart hvilket felt som inneholder en feil, hva som er galt og hvordan brukeren kan rette den. Unngå generelle meldinger som «Feil» eller «Ugyldig verdi». Bruk heller konkrete meldinger, for eksempel «E-postadressen må inneholde @». Se mønsteret for feilmeldinger i skjema for anbefalinger om når og hvordan feilmeldinger bør vises.

<span data-size="sm">
WCAG: [3.3.1 Identifikasjon av feil](https://www.uutilsynet.no/veiledning/331-identifikasjon-av-feil/1262) og [3.3.3 Forslag ved feil](https://www.uutilsynet.no/veiledning/333-forslag-ved-feil/1264)
</span>

<Divider/>

\
**Placeholder skal ikke brukes som eneste ledetekst eller til viktig informasjon.**

Placeholder forsvinner når brukeren begynner å skrive og blir ikke alltid lest opp av skjermlesere. Bruk derfor placeholder kun som et eksempel på forventet innhold eller format, ikke som en erstatning for ledetekst eller hjelpetekst.

<span data-size="sm">
WCAG: [3.3.2 Ledetekster eller instruksjoner](https://www.uutilsynet.no/veiledning/332-ledetekster-eller-instruksjoner/1263)
</span>

<Divider/>

\
**Bruk riktig autocomplete-verdi når feltet gjelder brukeren selv.**

Autocomplete gjør det enklere å fylle ut skjema ved å la nettleseren foreslå informasjon som navn, adresse, telefonnummer og e-post. Bruk dette når feltet gjelder informasjon om brukeren selv. Dersom feltet gjelder en annen person, bør autocomplete slås av eller ikke bruke personlige autocomplete-verdier.

<span data-size="sm">
WCAG: [1.3.5 Identifisere formål med inndata](https://www.uutilsynet.no/veiledning/135-identifisere-formal-med-inndata/1238)
</span>

<Divider/>

\
**Velg en input-type som passer til informasjonen brukeren skal oppgi.**

Riktig input-type gir bedre støtte fra nettleseren og hjelpemidler. Vær spesielt forsiktig med `type="number"`, som har semantikken *spinbutton* og bare bør brukes når brukeren skal angi en numerisk verdi som kan økes eller reduseres. Se veiledningen om input-typer for anbefalinger og vanlige fallgruver.

<span data-size="sm">
WCAG: [1.3.1 Informasjon og relasjoner](https://www.uutilsynet.no/veiledning/131-informasjon-og-relasjoner/1234) og [4.1.2 Navn, rolle, verdi](https://www.uutilsynet.no/veiledning/412-navn-rolle-verdi/1274)
</span>

<Divider/>

\
**Viktig informasjon skal ikke kun formidles med prefix eller suffix.**

Prefix og suffix blir ikke lest opp av skjermlesere. Dersom informasjonen er nødvendig for å forstå hva brukeren skal fylle inn, skal den også være en del av ledeteksten eller hjelpeteksten.

<span data-size="sm">
WCAG: [1.3.1 Informasjon og relasjoner](https://www.uutilsynet.no/veiledning/131-informasjon-og-relasjoner/1234)
</span>


## Ikke bruk deaktiverte felt

Ikke bruk deaktivert tilstand (`disabled`) på tekstfelt. Tenk heller over om du trenger å vise feltet i det hele tatt, eller om du heller kan skrive informasjonen ut i ren tekst eller bruke `readonly`.
