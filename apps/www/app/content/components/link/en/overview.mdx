**Use `Link` when**
- the user needs to navigate to another page within the solution
- the navigation leads to an external website
- a file should be downloaded

**Avoid `Link` when**
- the purpose is not navigation; use a [`Button`](/no/components/button/overview) instead
- the action changes the state of the solution, such as opening a dialog, starting a process or saving data

## Guidelines

It is important not to overuse links, as this can make it harder for users to find their way forward.

### Opening links in a new tab/window

Both internal and external links should open in the same tab/window.

## Text

The article [Labelling links](https://www.designsystemet.no/no/patterns/external-links#spr%C3%A5k) contains guidelines for how to write link text. It is important that link text is descriptive and gives users a clear indication of what to expect when they activate the link.
