When asking users for a date, simple text fields or the browser’s built-in support for date input often work best.

Instead of a custom date picker, we therefore provide guidance on [how to ask users for date and time](/en/patterns/date-and-time).

<Card data-color='info'>
Missing any functionality that native date input does not support? Please share your needs in the [date picker discussion on GitHub](https://github.com/digdir/designsystemet/discussions/1821).
</Card>
