## CSS variablar og data-attributter
Størrelser styres med [`data-size`](/no/fundamentals/code/sizes) og farger med [`data-color`](/no/fundamentals/code/colors). Komponenten vil arve nærmeste forelder med disse satt.

Checkbox er `ds-input` med `type="checkbox"` så variabler og data-attributter som gjelder for [input](/no/components/docs/input/overview) gjelder også for checkbox.

## React

I React er det enklere å bygge opp en checkbox, da vi har en ferdig komponent som tar seg av det meste av logikken og tilstandshåndteringen for oss.

`Checkbox` er ein samansatt komponent som bruker [field](/no/components/docs/field/overview), [label](/no/components/docs/label/overview), [validation message](/no/components/docs/validation-message/overview) og [input](/no/components/docs/input/overview) for å lage ein checkbox.

I tillegg har vi en egen [useCheckboxGroup](/no/components/utilities/use-checkbox-group) React-hook for å lettere håndtere en gruppe av Checkbox-komponenter.

```tsx
import { Checkbox } from '@digdir/designsystemet-react';

<Checkbox label="Checkbox label" value="value" />
```

### Indeterminate checkbox

Legg inn `allowIndeterminate: true` i `getCheckboxProps` fra [useCheckboxGroup](/no/components/utilities/use-checkbox-group) for å opprette en overordnet checkbox som kan velge eller fjerne alle alternativer. Dette aktiverer en ekstra tilstand, `indeterminate`, ved siden av `checked` og `unchecked`. Den vises med en horisontal strek når én eller flere checkbox er markert i gruppen. Den vises som en vanlig checkbox dersom alle i gruppen har samme tilstand.
