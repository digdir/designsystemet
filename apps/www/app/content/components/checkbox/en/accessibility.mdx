## What you need to check

**Groups of choices must have a descriptive legend.**

The legend, help text, and error messages must be connected to the group so that they are available to assistive technologies.

You can use [Fieldset](/en/components/docs/fieldset/overview) to connect the legend, help text, and error messages.

<span data-size="sm">
	WCAG: [1.3.1 Info and relationships](https://www.uutilsynet.no/veiledning/131-informasjon-og-relasjoner/1234)
</span>

<Divider/>

\
**The field must have a clear accessible name that matches the visible label.**

The accessible name must contain the visible text that describes the field. This allows users of speech recognition and screen readers to refer to the field using the same name shown on screen.

<span data-size="sm">
	WCAG: [2.5.3 Label in Name](https://www.uutilsynet.no/veiledning/253-ledetekst-i-navn/1256)
</span>

<Divider/>

\
**A checkbox without a visible label must have an accessible name.**

This applies, for example, when a checkbox is used in a table and the name must be derived from the content of the row or column.

The accessible name must make it clear what the checkbox refers to.

<span data-size="sm">
	WCAG: [4.1.2 Name, role, value](https://www.uutilsynet.no/veiledning/412-navn-rolle-verdi/1267)
</span>

<Divider/>

\
**Required fields must be clearly marked.**

Users must easily understand which fields must be filled in. Follow the [pattern for required and optional form fields](/en/patterns/required-and-optional-fields).

<span data-size="sm">
	WCAG: [3.3.2 Labels or Instructions](https://www.uutilsynet.no/veiledning/332-ledetekster-eller-instruksjoner/1263)
</span>

<Divider/>

\
**Error messages must be clearly linked to the correct field and explain how to fix the error.**

The error message must make it clear which field contains an error, what is wrong, and how the user can fix it. Avoid generic messages such as “Error” or “Invalid value”. Use specific messages instead, for example “The email address must contain @”. See the [pattern for user-triggered error messages](/en/patterns/errors) for recommendations on when and how error messages should be shown.

<span data-size="sm">
	WCAG: [3.3.1 Error Identification](https://www.uutilsynet.no/veiledning/331-identifikasjon-av-feil/1262) and [3.3.3 Error Suggestion](https://www.uutilsynet.no/veiledning/333-forslag-ved-feil/1264)
</span>
