## Implementation tests

**1. An informative `Badge` showing important information (for example a count) must be included in the interactive element’s accessible name**

WCAG: [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90), [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva-a/98), [2.4.4](https://www.uutilsynet.no/wcag-standarden/244-formal-med-lenker-niva-aa/112)

**2. If a `Badge` is placed outside the element, the badge information must be included in the element’s accessible name.** 

WCAG: [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90), [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva-a/98)

**3. A decorative `Badge` that conveys no information must be hidden from screen readers**

WCAG: [1.1.1](https://www.uutilsynet.no/wcag-standarden/111-ikke-tekstinnhold-niva/22)

**4. A `Badge` that changes value (for example 1 → 2) must be announced to screen readers**

WCAG: [4.1.3](https://www.uutilsynet.no/wcag-standarden/413-statusbeskjeder-niva-aa/152)

**5. A `Badge` must not rely solely on color or position to convey information**

WCAG: [1.4.1](https://www.uutilsynet.no/wcag-standarden/141-bruk-av-farge-niva/93)
