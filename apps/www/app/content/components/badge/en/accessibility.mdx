## Implementation tests

**1. Informative `Badge` must be included in the programmatic name**

- A `Badge` that conveys important information (for example the number of unread messages) must be included in the programmatic name of the interactive element it belongs to.

WCAG: [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90), [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva/121), [2.4.4](hhttps://www.uutilsynet.no/wcag-standarden/244-formal-med-lenke-i-kontekst-niva/106)

**2. Badge information must be programmatically associated with the control**

- If a `Badge` is placed outside the interactive element, the information must be programmatically associated with the control.
- Solutions where a `Badge` is not part of the control’s programmatic name should be avoided.

WCAG: [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90), [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva/121)

**3. Decorative `Badge` must be hidden from screen readers**

- A `Badge` that is purely decorative must be hidden from screen readers (`aria-hidden="true"`).

WCAG: [1.1.1](https://www.uutilsynet.no/wcag-standarden/111-ikke-tekstlig-innhold-niva/87)

**4. Dynamically updated `Badge` must be announced**

- If a `Badge` is updated dynamically (for example from 1 to 2), the change must be announced to screen readers in an appropriate manner.

WCAG: [4.1.3](https://www.uutilsynet.no/wcag-standarden/413-statusbeskjeder-niva-aa/152)

**5. `Badge` must not rely on color alone**

- Information conveyed by a `Badge` must be understandable without relying solely on color or position.
- A visual indicator (for example a red dot) alone is not sufficient.

WCAG: [1.4.1](https://www.uutilsynet.no/wcag-standarden/141-bruk-av-farge-niva/93)
