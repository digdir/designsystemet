## Implementation tests

**1. Informative `Avatar` must have a programmatic name**

- An `Avatar` that represents an identity must have a programmatic name that corresponds to that identity.
- If multiple avatars are displayed together, each informative `Avatar` must have a unique programmatic name.

WCAG: [1.1.1](https://www.uutilsynet.no/veiledning/111-ikke-tekstlig-innhold/1231), [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90)

**2. Decorative `Avatar` must be hidden from screen readers**

- A `Avatar` that does not convey unique information must be hidden from screen readers (`aria-hidden="true"`).

WCAG: [1.1.1](https://www.uutilsynet.no/veiledning/111-ikke-tekstlig-innhold/1231)

**3. Clickable `Avatar` must have a clear programmatic name**

- A clickable `Avatar` must have a programmatic name that clearly describes the action (for example “Profile for User Name”).

WCAG: [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva/121), [2.4.4](https://www.uutilsynet.no/wcag-standarden/244-formal-med-lenke-i-kontekst-niva/106)

**4. Clickable `Avatar` with initials must support voice control**

- If a clickable `Avatar` visually displays initials, those initials should be included in the programmatic name.

WCAG: [2.5.3](https://www.uutilsynet.no/wcag-standarden/253-ledetekst-i-navn-niva/150)

## How to describe `Avatar` for screen readers

`Avatar` must either have an `aria-label` that describes who or what the avatar represents, or be set to `aria-hidden` with visible text conveying the same information. Do not state in the `aria-label` that it is an image - screen readers announce this automatically. It is sufficient to use the name, for example `aria-label="Ola Nordmann"`.

Images and other `children` are automatically given `aria-hidden="true"` to prevent information from being repeated.
