## Implementation tests

**1. Informative `Avatar` must have a programmatic name, and each name must be unique if multiple avatars are shown**

- If multiple avatars are displayed together, each programmatic name must be unique.

WCAG: [1.1.1](https://www.uutilsynet.no/wcag-standarden/111-ikke-tekstinnhold-niva/22), [1.3.1](https://www.uutilsynet.no/wcag-standarden/131-informasjon-og-relasjoner-niva/90)

**2. Decorative `Avatar` that conveys no unique information must be hidden from screen readers**

WCAG: [1.1.1](https://www.uutilsynet.no/wcag-standarden/111-ikke-tekstinnhold-niva/22)

**3. Clickable `Avatar` must have a clear programmatic name**

WCAG: [4.1.2](https://www.uutilsynet.no/wcag-standarden/412-navn-rolle-verdi-niva-a/98), [2.4.4](https://www.uutilsynet.no/wcag-standarden/244-formal-med-lenker-niva-aa/112)

**4. Clickable `Avatar` with initials (for example “ON” for Ola Nordmann) must include the initials in its programmatic name**

WCAG: [2.5.3](https://www.uutilsynet.no/wcag-standarden/253-ledetekst-i-navn-niva-aa/126)

## How to describe `Avatar` for screen readers

`Avatar` must either have an `aria-label` that describes who or what the avatar represents, or be set to `aria-hidden` with visible text conveying the same information. Do not state in the `aria-label` that it is an image - screen readers announce this automatically. It is sufficient to use the name, for example `aria-label="Ola Nordmann"`.

Images and other `children` are automatically given `aria-hidden="true"` to prevent information from being repeated.
