Det er vanlig å bli forvirret mellom `Fieldset` og `Field`.
En god huskeregel er at **`Fieldset` er et sett med `Field`**

<Story story="Preview" />
<ReactComponentDocs />

## Bruk

```tsx
import { Fieldset, Radio } from '@digdir/designsystemet-react';

<Fieldset>
  <Fieldset.Legend>Spørsmål</Fieldset.Legend>
  <Fieldset.Description>Beskrivelse</Fieldset.Description>
  <Radio label="Alternativ 1" name="radio" value="1" />
  <Radio label="Alternativ 2" name="radio" value="2" />
</Fieldset>
```

## Eksempel

### Med checkbox

Både [`Checkbox`](/components/checkbox/code) og [`Radio`](/components/radio/code) bør grupperes i et `Fieldset`, selv om det kun er ett element i gruppen.

### Legend som heading

Dersom det er en side som kun har skjema på seg, og tittel på siden (`<h1>`) er samme som vil være i `<legend>`, kan du legge `<h1>` inni `Fieldset.Legend`.

<Story story="LegendAsHeading" />

## HTML

Klassenavnet `ds-fieldset` legges på `<fieldset>`-elementet.

```html
<fieldset class="ds-fieldset">
  <legend class="ds-label" data-weight="medium">Godtar du vilkårene?</legend>
  <!-- dine fields -->
</fieldset>
```
