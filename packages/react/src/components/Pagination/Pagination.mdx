# Pagination

`Pagination` er en liste med knapper som brukes for å navigere mellom ulike sider med innhold, for eksempel søkeresultater eller tabeller.

<Primary />
<Controls of={PaginationStories.Preview} />

## Bruk

`Pagination` er en kontrollert komponent. Det vil si at komponentens tilstand om hvilken side som er aktiv styres utenfra.

```tsx
import { Pagination } from '@digdir/designsystemet-react';

const onChange = (event, page: number) => console.log(`Going to page ${page}`);
const [currentPage, setCurrentPage] = useState(4);
const { pages, prevButtonProps, nextButtonProps, hasNext, hasPrevious } = usePagination({
  currentPage,
  setCurrentPage,
  onChange,
  totalPages: 10,
  showPages: 7,
});

<Pagination>
  <Pagination.List>
    <Pagination.Item>
      <Pagination.Button aria-label='Forrige side' {...prevButtonProps}>
        Forrige
      </Pagination.Button>
    </Pagination.Item>
    {pages.map(({ page, itemKey, buttonProps }) => (
      <Pagination.Item key={itemKey}>
        {typeof page === 'number' && (
          <Pagination.Button {...buttonProps} aria-label={`Side ${page}`}>
            {page}
          </Pagination.Button>
        )}
      </Pagination.Item>
    ))}
    <Pagination.Item>
      <Pagination.Button aria-label='Neste side' {...nextButtonProps}>
        Neste
      </Pagination.Button>
    </Pagination.Item>
  </Pagination.List>
</Pagination>
```
## Eksempler

### Paginering med `usePagination`

Bruk `usePagination` for paginering-logikk sammen med de enkelt del-komponentene.
Les mer om `usePagination` [her](/docs/utilities-usepagination--docs).

### Paginering med lenker
Vær oppmerksom på at knappene er som standard `button` elementer. Hvis `Pagination` skal navigere til en ny URL, må knappene være lenker `<a>`. 

Alle del-komponenter støtter `asChild` som kan brukes til å endre `Pagination.Button` til lenker (`a`).

### Responsiv visning
På små skjermer er det lurt å begrense antall synlige sidetall for å spare plass. Vis kun:
- den aktive siden
- én side før og etter
- første og siste side

## Retningslinjer
Bruk `Pagination` når innholdet er for stort for én side, for eksempel ved søkeresultater eller lange lister, og brukeren trenger å navigere frem og tilbake. Vi deler kun opp innhold i flere sider når det forbedrer lastetiden eller brukervennligheten. 

**Passer til å**
- navigere mellom relaterte sider eller deler
- dele inn søkeresultater i flere sider
- dele inn store mengder innhold i flere sider, for eksempel i tabeller

**Passer ikke til å**
- dele inn et skjema i steg, da bør du heller bruke en stegindikator
- vise fremdrift

## Tekst
Vi bruker et fast mønster for `Pagination` med ord: «Forrige» og «Neste». 

Unngå «Forrige side», «Neste side», «Gå tilbake», «Gå videre», «Videre» og «Tilbake».
